# 产品缺陷深度审阅（体验 / 使用 / 习惯）

> 审阅范围：当前 `kaoshi` 全栈工程（学生作答、草稿、同步、教师发布、批改、家长、多班）。
> 方法：逐文件读源码 + 对照 `完整版需求PRD.md`，缺陷均附代码定位，非凭空判断。
> 严重度：🔴P0 阻断/数据错误 ｜ 🟠P1 体验断裂/高频痛点 ｜ 🟡P2 习惯/打磨。

---

## 一、体验（Experience）

### 🔴 P0-1 两条"发布作业"入口，数据模型不一致
- **现象**：教师从 `TeacherDashboard` 点"发布作业"走 `api.post('/api/exams')` → `createExam`；从 `PaperLibrary` 点"发布"走 `assignmentsApi.publish` → `publishAssignment`。
- **代码**：`src/components/dashboard/TeacherDashboard.tsx:225` 调 `/api/exams`；`server/controllers/exams.ts:26` 的 `createExam` 只写 `exams`，**不写 `assignments`、不带 `timePolicy`/`allowRedo`/`theme`/`assignmentId`**。而 `server/controllers/assignments.ts:221-224` 的发布**同时**写 `exams`（带 `assignmentId`）+ `assignments`。
- **影响**：同一 UI 内两条路产出的作业"长相不同"——用 Dashboard 发的作业**永不限时、永不迟交、不可配置重做、无主题**；用预存库发的才有。学生端 `getStudentExams` 只读 `exams`（含/不含 `timePolicy` 取决于入口），体验不可预期。
- **建议**：收敛为单一发布入口（统一走 `assignments`），`createExam` 旧路由下线或仅内部复用。

### 🔴 P0-2 跨设备作答/提交会"丢页"（手动同步是推非拉 + 提交整体覆盖）
- **现象**：学生设备A答第2页并"同步到云端"，设备B答第1页并"同步到云端"——云端 `mergeAnswers` 两页都在；但若任一侧**提交**，`submitExam` 对已提交答卷执行 `submission.answers = answers`（整体覆盖，非 merge，见 `server/controllers/exams.ts:265`）。后提交者覆盖前者，先写的那几页凭空消失。
- **影响**：低龄生天然会在平板上、家长手机上交替写，跨设备交卷丢页是**静默数据错误**，老师看到的答卷不完整。
- **建议**：提交也走 `mergeAnswers`（按页合并）而非整体覆盖；或引入版本号/冲突检测，提交前提示"云端有更新，是否合并"。

### 🟠 P1-3 本地草稿"退出即丢最后几笔"，且无防关页拦截
- **代码**：自动存盘仅"落笔停顿 4s"或"翻页"触发（`StudentWorkspace.tsx:177-184` `setTimeout(4000)`）；**无 `beforeunload` 拦截**。学生写完最后一笔立刻点"回大厅"或关浏览器 → 4s 防抖未触发 → 末笔丢失。
- **影响**：手写场景下一页常 10s 内画完即交/即退，丢失概率高，且与"草稿永不丢失"的承诺相悖。
- **建议**：翻页/交卷/退出前同步一次；加 `beforeunload` 兜底；或落笔即防抖缩短至 1.5s。

### 🟠 P1-4 手动"同步到云端"是反习惯的，且零引导
- **现象**：跨设备续写依赖用户主动点同步（`StudentWorkspace.tsx:154` `syncToCloud`）。低龄/家长**无"云端同步"心智**，答完直接交，从不点同步；换设备发现"上次写的没了"。
- **影响**：本地 IndexedDB 随浏览器/origin，清缓存、换 Safari/Chrome、平板还原即全灭；未同步则云端也无，且**无任何"您有未同步草稿"的强提醒**。
- **建议**：至少每次离开工作区自动尝试一次静默同步（失败再提示）；大厅对"本地有、云端无"的草稿打醒目徽标"⚠️仅本机"。

### 🟠 P1-5 仍大量使用原生 `confirm`/`alert`，违反 PRD
- **代码**：`StudentWorkspace.tsx:112` 云端恢复 `window.confirm`、`:170/:275/:280` `alert`。
- **影响**：童趣 UI 突然弹系统级灰框，风格割裂；Pad/移动端原生 confirm 常被折叠不可点；PRD 第7节明确要求"统一 toast/错误边界替代 alert"。
- **建议**：引入全局 toast + 自定义确认弹层（卡通风格）。

### 🟠 P1-6 严格限时到点"直接收白卷"，无缓冲
- **代码**：`StudentWorkspace.tsx:223-229` 到点 `handleSubmit(true)` 自动提交，跳过确认。若学生一页未写 → 提交空卷（`status=submitted`），且 `submitExam` 对已提交默认放行重做仅当 `allowRedo` 且老师 reopen。
- **影响**：低龄生误触/走神被强制交白卷，且 Dashboard 发布的作业 `allowRedo` 永不生效（见 P0-1），**无法挽回**。
- **建议**：到点前 10s 醒目倒数 + 一次确认；空卷拦截"你一页都没写，确定交吗？"。

### 🟡 P2-7 空卷可被成功提交
- **代码**：`submitExam` 校验仅 `!answers || !Array.isArray(answers)`（`exams.ts:220`），空数组 `[]` 通过；`filter(canvasData!=='')` 后仍可能为空，但照常提交。
- **建议**：提交前至少拦截"0 页有效笔迹"。

### 🟡 P2-8 批改后学生不可重做，与 PRD 矛盾
- **代码**：已 `graded` 的提交，`saveDraft`/`submitExam` 均 400 拦截（`exams.ts:185`、`:254`）；PRD 第4.4节要求"隐藏红笔让学生错题重做"。
- **影响**：老师批改完，学生想订正却进不去。"重开"仅作用于 `assignments`（`reopenAssignment`），而 Dashboard 发布的 exam 无 `assignmentId` → 重开无效（叠加 P0-1）。
- **建议**：增加"订正模式"：graded 后允许学生重做（新建一份 submission 或标记 redo）。

## 二、使用（Usage，教师/运营视角）

### 🟠 P1-9 无批量加学生，且"并入已有学生"需手记学号
- **现象**：教师逐个填表单加人；"已有学生并入另一班"需勾选"已有模式"再手输学号——**无下拉搜索/选择**。
- **影响**：30 人班点 30 次；把学生加第二班还得背下学号。真实使用最大痛点之一。
- **建议**：加"粘贴学号/Excel 导入"批量入口；"并入"提供本班外学生下拉搜索。

### 🟠 P1-10 无"退班/移出/转班"功能
- **代码**：`addStudentToClass` 只 `studentIds.push`（`classes.ts`），无 remove；升班靠手动"并入"，源班不自动清空。
- **影响**：学生转走/退班后名册永远留着，历史作业仍可见、无法清理；教师端无"转出/转入"按钮。
- **建议**：补"移出班级"；升班提供"一键转班"（源班移除 + 目标班加入）。

### 🟡 P2-11 迟交扣分规则未落地
- **代码**：`evaluateSubmitTime` 只标 `isLate`/`lateMs`，PRD 第5节要求"每超 1h 扣 X 分"可配置，代码无扣分。
- **建议**：在 `timePolicy` 加 `latePenalty`，提交/批改时计入。

### 🟡 P2-12 教师无"提交/迟交"主动通知
- **现象**：学生提交、迟交后教师无红点/推送，只能手动刷列表。
- **影响**：习惯上教师会漏批改，迟交名单雪藏。
- **建议**：大厅"待批改"角标 + 新提交提示音/红点。

## 三、习惯（Habits，用户心智/日常行为）

### 🟠 P1-13 家长 `childId` 单值，二孩家庭无法用
- **代码**：`User.childId?: string`（`db.ts:13`）单字段；`getSubmissionDetails` 按 `parent.childId === studentId` 校验（`exams.ts:339`）。
- **影响**：二孩同校不同班极常见，一个家长账号只能绑一个孩子，另一个孩子的成绩看不到。
- **建议**：改为 `childIds: string[]`。

### 🟡 P2-14 多设备并发无锁，互相覆盖
- **现象**：设备A、B都从云端恢复同一草稿 → 都改 → 都提交 → 后者覆盖（整体覆盖，P0-2）。
- **建议**：提交带 `baseVersion`；冲突时提示用户"检测到他人已提交，是否仍覆盖"。

### 🟡 P2-15 首登无强制改密、无找回
- **代码**：学生用老师给的初始密码登录，无改密/找回流程。
- **影响**：低龄生记不住密码 → 反复找老师重置，运营负担高。
- **建议**：首次登录强制改密；提供"学号+绑定码"自助找回。

### 🟡 P2-16 "本地草稿 vs 云端草稿"心智不统一
- **现象**：学生端草稿箱读本地 IndexedDB 显示"草稿中"（`StudentDashboard`），但换设备看不到；"到底以哪个为准"学生/家长说不清。
- **建议**：统一以"是否有未交答案"为状态，本地/云端只作为存储位置标签，不并列成两种"草稿中"。

## 修复优先级建议
1. **P0-1 收敛发布入口**（否则限时/重做/主题功能随入口漂移，后端逻辑半失效）
2. **P0-2 提交改按页 merge + 版本冲突**（静默丢页是数据正确性事故）
3. **P1-3/4 退出兜底存盘 + 离开自动静默同步**（兑现"草稿不丢"承诺）
4. **P1-5 原生弹窗换 toast/自定义确认**（Pad 体验与 PRD 合规）
5. **P1-9/10 批量加人 + 退班/转班**（教师日常最高频痛点）
6. **P1-13 家长 childIds 多孩**（二孩家庭硬伤）
7. 余下 P2 打磨项随版本迭代。
