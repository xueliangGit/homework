# 儿童考试系统 · 上线与体验路线图

> 由**产品专家「童童」**（儿童教育产品体验设计师，分类 01-ProductDesign）与**技术专家「阿码」**（全栈技术架构师，分类 02-Engineering）双视角共同制定。
> 目标：让「快乐作业考试本」达到 **① 能正常运行并显示（可上线）** 与 **② 儿童 / 教师 / 家长三端交互舒服**。

---

## P0 — 必须先行（挡住运行 / 上线）

### 🛠 技术（阿码）
- [ ] **配置并验证 Tailwind**：安装 `tailwindcss` + `@tailwindcss/postcss`（或等价方案）；建 `tailwind.config.js` 扫描 `src`；在 `index.css` 加 `@tailwind` 指令；确保 `npm run build` 通过且工具类生效。
  - 验收：`npm run build` 成功；`dev` 下界面布局正常、不再塌陷（当前 `dist` CSS 仅 8KB 且无任何工具类，是让项目能看的第一要务）。
- [ ] **核对依赖可安装性**：`vite: ^8`、`lucide-react: ^1.16.0`、`typescript: ~6` 等异常版本需先核对锁文件与真实可安装性，再决定升/降。
- [ ] **复现运行**：`npm install && npm run dev`，建立当前可运行基线。

### 🎨 产品（童童）
- [ ] **建立三角色体验地图**：教师（发卷→批改→打分）/ 学生（书写→提交→错题重做）/ 家长（绑定→查分→成长明信片），标注当前 happy path 与断点。

---

## P1 — 上线前加固（安全 / 可靠性）

### 🛠 技术（阿码）
- [ ] **安全加固**：
  - JWT 密钥外置 `.env`（当前硬编码 `'cute-kaoshi-system-secret-key-123456'`）；
  - CORS 限定来源（当前 `cors()` 全开）；
  - 家长绑定校验（教师下发绑定码，防 A 家长绑 B 孩子）；
  - 提交接口校验班级归属（防学生提交到任意试卷）；
  - 收敛 `express.json({limit:'50mb'})` 并加速率限制。
- [ ] **存储升级**：修复 `JSONDatabase.init()` 竞态（单例 + 懒加载锁）；响应排除大体积 `pages` 字段；给出从单进程 JSON 到 SQLite/Postgres 的演进方案。

### 🎨 产品（童童）
- [ ] **PRD 落差核对**（对照 `PRD_ENHANCEMENT.md`）：草稿防丢（IndexedDB 每 10s 备份）、软删除撤回（5 分钟 + 队列）、Pinch-Zoom（双指缩放/平移 + 弹性回弹）、能力雷达图报告卡 —— 排优先级。
- [ ] **统一反馈与空态**：用 toast / 错误边界替代全局 `alert`；关键操作加确认与防误触。

---

## P2 — 交互舒服（体验打磨）

### 🎨 产品（童童）
- [ ] **儿童友好基础**：点击热区 ≥ 48×48px、动效反馈、撒花与成就体系；移动端 Touch 顺滑。
- [ ] **进阶交互**：触觉反馈 `navigator.vibrate`、双指缩放平移、弹性回弹。
- [ ] **成长明信片**：导出 1080×1920 + 能力雷达图 + 防伪分享码（当前为 1000×700 无雷达图）。

### 🛠 技术（阿码）
- [ ] **Canvas 优化**：书写 / 橡皮 / 撤销算法打磨；引入路由（React Router）替代 `view` 字符串状态机。
- [ ] **部署上线**：CloudStudio / 静态托管发布，给出可访问地址；前端静态化 + 后端独立进程。

---

## 建议推进顺序
**P0（Tailwind + 复现） → P1（安全 + 存储） → P2（交互 + 功能） → 部署上线**

> 备注：当前项目已存有 `PRD_ENHANCEMENT.md`（儿童向 UX 规范）与三位一体角色设计，是双专家工作的核心参照。
