# 完整版需求规格（PRD v2）—— 全学段手写作业与考试系统

> 定位升级：从「儿童趣味版」扩展为 **幼 / 小 / 中 全学段通用** 的在线手写作业与考试系统。
> Pad 优先（触控笔手写体验最佳），主题可切换，教师预存试卷、按需发布作业，时限策略可选，全链路留痕可查。
> 本文件基于现有工程（已具备 Tailwind、Canvas 手写、鉴权、安全加固、存储竞态修复）做能力扩展，不推倒重来。

---

## 0. 一句话目标

让教师「先备卷、后发布」，学生在 Pad 上像在本子上一样手写作答，系统按可配置的时限策略收卷，教师红笔批改、家长查看成长，每一步操作都可追溯、可审计、可导出。

---

## 1. 角色与全链路（见链路图）

| 环节 | 角色 | 关键动作 | 留痕 |
|---|---|---|---|
| 建卷 | 教师 | 在「试卷库」创建/手绘/上传试卷，存为预存草稿 | 建卷日志 |
| 发布 | 教师 | 从试卷库选卷 → 配置班级/时限/主题 → 发布为作业 | 发布日志 |
| 作答 | 学生 | Pad 手写作答（多页），草稿自动保存 | 进入/提交时间、设备 |
| 提交 | 学生 | 按时提交 或 逾期提交（依策略标记） | 提交时间、用时、是否迟交 |
| 批改 | 教师 | 红笔批注 + 评分 + 评语 | 批改时间、批改人、改分记录 |
| 查看 | 家长 | 绑定孩子，看成绩与成长报告 | 查看日志 |

## 2. 设备与手写交互（Pad 优先）

现有 `CanvasDraw` 已支持触摸、双指缩放/平移、触觉反馈。需进一步强化为「答卷级」体验：

- **电容笔优先**：改用 Pointer Events，读取 `pressure` 做笔锋粗细；区分 `pointerType`(pen/touch/mouse)，笔尖细、手指粗。
- **防误触（手掌抑制）**：检测到手掌大面积接触时忽略，仅笔尖落墨。
- **书写手感**：贝塞尔平滑（已有）、笔锋、橡皮（局部擦除，已有）、撤销/重做、清空、多页翻页。
- **响应式适配**：
  - Pad：横屏/竖屏自适应，画板占满可用区，工具栏悬浮 / 侧边；
  - 手机：画板可缩放（已有双指），工具栏底部抽屉；
  - 桌面：鼠标书写回退，Ctrl+滚轮缩放（已有）。
- **离线草稿**：作答进度本地持久化，断网可继续写，联网自动同步。

## 3. 主题系统（可切换）

视觉由 CSS 变量驱动（`src/styles/variables.css`），运行时切换 `<html data-theme="...">`，用户偏好持久化。

- **学段主题（3 套基线）**：
  - `you`（幼）：活泼卡通、圆润、高饱和、大字大按钮；
  - `xiao`（小）：明亮糖果、亲和；
  - `zhong`（中）：清爽简约、克制、信息密度高。
- **学科叠加（可选）**：语文 / 数学 / 英语 / 综合 等，在主色调上叠加配色与图标氛围。
- **切换入口**：
  - 教师发布作业时可指定该作业的学段/学科主题；
  - 学生/家长端可在「设置」里切换自己偏好的显示主题（不影响作业内容）。
- **实现要点**：所有颜色/圆角走变量；禁止硬编码色值；主题即一组变量映射。

## 4. 核心功能模块

### 4.1 试卷预存库（Paper Library）【新增】
教师把卷子先存起来，随时取用。与「已发布作业」分离。

- 创建方式：
  - 空白画布手绘（多页）；
  - 上传图片（PNG/JPG）按页拼成试卷；
  - 上传 PDF 自动按页拆图（已有 `pdfParser`，需接入）；
  - （可选）从模板库新建。
- 管理：列表 / 缩略图预览 / 编辑 / 复制 / 移动分类 / 软删除（回收站可恢复）。
- 分类与检索：按学段、学科、年级、标签、关键词搜索。
- 数据：`papers` 集合（独立于 `exams`/作业）。

### 4.2 发布作业（Assignment）
从预存库选卷 → 配置 → 一键发布，学生端即时可见。

- 配置项：
  - 来源试卷（来自 4.1）；
  - 目标：班级 / 指定学生；
  - 标题、说明（给学生的提示）；
  - **时限策略（详见第 5 节）**；
  - 主题（学段/学科）；
  - 是否允许重做、最多重做次数；
  - 发布时间（立即 / 定时）。
- 发布后生成「作业」实体，学生端出现在「待完成」列表。

### 4.3 学生作答
- 作业列表（待完成 / 进行中 / 已提交 / 已批改）；
- 倒计时 / 截止提醒（依时限策略）；
- 手写作答（多页，见第 2 节）；
- 草稿自动保存（每 10s，IndexedDB，PRD 原要求）；
- 提交：按时提交 或 依策略允许逾期提交（标记「迟交」）。

### 4.4 教师批改
- 三层叠加（原卷 + 学生笔迹 + 红笔，已有）；
- 红笔手写批注、打分、评语、错因标签；
- 支持「隐藏红笔」让学生错题重做（已有）；
- 批量批改、错题归集。

### 4.5 家长
- 绑定（绑定码防串绑，已有）；
- 查看孩子成绩、提交状态、迟交标记；
- 成长报告（升级为 1080×1920 + 能力雷达图 + 防伪分享码，待做）。

### 4.6 记录与可查（Audit / Traceability）【重点】
目标：每一步都可追溯，教师/管理员可在「记录中心」查询导出。

- **操作日志 `operationLogs`**：谁(user)、角色、动作(建卷/发布/提交/批改/改分/删除)、对象 id、时间、IP、设备类型。
- **提交记录**（写入 `submissions`）：提交时间、首入时间、用时、是否迟交、迟交时长、设备类型、页数、IP。
- **成绩记录**（写入 `grades`/批改结果）：分数、评语、批改时间、批改人；改分留痕（原值→新值）。
- **记录中心 UI**：按 学生 / 作业 / 时间范围 / 动作类型 / 是否迟交 多维筛选；支持导出 Excel / PDF。

## 5. 时限策略（详细）

发布作业时二选一（可组合）：

| 模式 | 行为 | 学生端表现 | 记录 |
|---|---|---|---|
| **A. 严格限时** | 到达截止时间系统自动锁定，禁止提交 | 按钮置灰「已截止」 | 未提交者标记「未交」 |
| **B. 宽限迟交** | 截止后仍可作答并提交 | 提示「已逾期，仍可提交」 | 标记「迟交」+ 迟交时长 |
| （可选）倒计时 | 设定作答时长（如 30 分钟），到点自动收卷 | 实时倒计时 | 超时自动提交当前内容 |

- 截止方式可选：**截止日期时间** 或 **倒计时时长**（两者可并存：先到者生效）。
- 迟交是否扣分 / 扣分规则：可配置（如每超 1 小时扣 X 分，或仅标记不扣分）。
- 学生端实时显示剩余时间 / 已逾期状态；教师端作业详情显示提交率、迟交名单。

## 6. 数据模型（概览，向后兼容现有）

| 集合 | 关键字段 | 说明 |
|---|---|---|
| `papers` | id, title, ownerId, segment(you/xiao/zhong), subject, tags, pages[], status(draft), createdAt | 预存试卷库（新增） |
| `exams`→`assignments` | id, paperId, title, classIds, timePolicy{ mode, deadline, durationMin, latePenalty }, theme, allowRedo, status, createdAt | 已发布作业（扩展时限/主题） |
| `submissions` | id, assignmentId, studentId, pages[], submittedAt, firstEnteredAt, durationMs, isLate, lateMs, device, ip | 作答（补留痕字段） |
| `grades` | id, submissionId, score, comment, teacherId, gradedAt, redrawPages[] | 批改（已有，补记录） |
| `operationLogs` | id, userId, role, action, targetType, targetId, at, ip, device | 全链路审计（新增） |
| `users/classes/themes` | 现有 + theme 偏好字段 | — |

> 注：为兼容现有数据，`exams` 可平滑升级为 `assignments`（增加字段，旧数据补默认值），不强制迁移。

## 7. 非功能要求

- **Pad 性能**：笔迹渲染 60fps，大图不卡顿（分页加载/离屏）。
- **离线可用**：草稿本地存，弱网可答。
- **安全**：沿用已做加固（JWT 外置、CORS 白名单、限流、绑定码、提交归属校验、分档 body 上限）。
- **可部署**：前端静态化 + 后端独立进程（见路线图「部署上线」）。
- **可维护**：路由化（后续引入 React Router）、统一 toast/错误边界替代 alert、关键路径单测。

## 8. 决策点（已确认 ✅）

1. **时限语义**：**两种都要** —— 截止日期时间 + 倒计时时长（先到者生效）。
2. **主题粒度**：3 套学段主题 + 学科叠加 **足够**，不做额外自定义配色。
3. **预存卷来源**：**三种都要** —— 手绘 / 上传图片 / PDF 导入。
4. **记录导出**：**Excel + PDF 都要**。
5. **数据持久化**：本轮 **直接转 SQLite**（替换 JSON 文件库）。
